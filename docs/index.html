Spielsysteme

Alles was ein gutes
Wirtschaftsspiel braucht.

Tick-basierter Loop, NPC-Konkurrenten, Tech-Tree, PvP — durchdacht aufgebaut auf Spring Boot und PostgreSQL.

⏱️
Tick-System
Alle 6h wird die Welt verarbeitet. KPIs, Gehälter, Events, NPC-Wachstum, Projekt-Fortschritt.
Core Loop
🤖
NPC-Konkurrenten
3 Persönlichkeiten: Aggressiv (+8%/Tick), Konservativ (+2%), Innovativ (volatil). Spawnen automatisch beim Start.
Lebendige Welt
🔬
Tech-Tree
15 Nodes in 3 Säulen: Operations, R&D, People. Research Points passiv pro Tick. 1 Node pro 24h.
Progression
🗡️
Sabotage
4 Aktionen: Leak, Hiring War, Fake PR, Infra-Angriff. Backfire-Mechanik basierend auf Ziel-Reputation.
PvP
📋
30 Projekte
6 pro Markt, hardcoded in Java. Risiko-Roll mit 4 Outcomes. Schalten Märkte, Rollen und Nodes frei.
Content
🌟
Prestige-System
Hard Reset mit Legacy-Multiplikator oder Soft Prestige (IPO, Markt-Dominanz). 35 permanente Achievements.

1× pro Run nutzbar

Quickstart

In 3 Schritten live.

Docker Compose übernimmt PostgreSQL, Flyway und den Bot in einem Befehl.

Docker
Lokal / Dev
.env Config
# 1. Repository klonen
git clone https://github.com/Fabsi07/OMIEXX.git
cd OMIEXX

# 2. Konfigurieren
cp .env.example .env
# DISCORD_TOKEN und ADMIN_DISCORD_IDS in .env eintragen

# 3. Starten
docker compose up -d

# Bot ist online. /start im Discord-Server eingeben.
# PostgreSQL starten
docker run --name omniexx-db \
  -e POSTGRES_DB=omniexx \
  -e POSTGRES_USER=omniexx \
  -e POSTGRES_PASSWORD=secret \
  -p 5432:5432 -d postgres:16

# Umgebungsvariablen setzen
export DISCORD_TOKEN=dein_token
export ADMIN_DISCORD_IDS=deine_discord_id

# Tick für Dev auf 1 Minute setzen
export TICK_INTERVAL_MS=60000

# Bot starten
./mvnw spring-boot:run
DISCORD_TOKEN=dein_bot_token
DB_URL=jdbc:postgresql://localhost:5432/omniexx
DB_USER=omniexx
DB_PASSWORD=secret

# Deine Discord-ID (Rechtsklick auf Profil → ID kopieren)
ADMIN_DISCORD_IDS=123456789012345678

# Tick-Interval: 6h = 21600000ms | Dev = 60000
TICK_INTERVAL_MS=21600000

# Optional: Channel-IDs für automatische Posts
LEADERBOARD_CHANNEL_ID=
EVENT_LOG_CHANNEL_ID=
